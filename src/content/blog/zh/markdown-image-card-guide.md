Markdown 图片卡片，是由结构化文本生成的小型视觉资产。它通常包含一个标题、几个短段落或列表，并通过排版让内容更容易扫读。

它有用的原因很简单：很多有价值的信息一开始都是纯文本，比如笔记、AI 回复、检查清单、提示词、代码示例或简短说明。把这些内容做成干净图片卡片，更适合发在社交平台、群聊、Newsletter、内部更新或文档讨论里。

## 好的图片卡片应该是什么样

好的图片卡片是聚焦的。它应该讲清楚一个想法，而不是逼读者打开一篇长文。

通常包含：

- 一个清晰标题
- 一个核心观点
- 简短段落
- 容易扫读的列表
- 有用的小标题
- 不包含私人上下文
- 不依赖多余装饰

弱卡片往往试图塞太多内容。如果图片变成一大段文字，它就不再是卡片，而更像文档截图。

## 适合做成 Markdown 卡片的内容

Markdown 图片卡片适合：

- 检查清单
- 提示词公式
- 简短教程
- 对比表格
- 发布说明
- 会议结论
- AI 回复总结
- 代码片段
- 引用卡片
- 工作流提醒

不太适合长文章、复杂文档、法律文本，或需要频繁更新的信息。

## 一个可复用 Markdown 卡片模板

不知道从哪里开始时，可以用这个结构：

```markdown
# 清晰标题

一句话说明这件事为什么重要。

## 关键点

- 第一点
- 第二点
- 第三点

## 下一步

做一个具体动作。
```

这个模板有效，因为它给读者一条路径：标题、原因、关键点、下一步。

## 示例：提示词公式卡片

原始想法：

```text
我想解释如何写出更好的 AI 视频提示词。
```

整理后的 Markdown：

```markdown
# AI 视频提示词公式

只写一个清晰镜头：

- 主体
- 动作
- 场景
- 镜头运动
- 光线
- 视觉风格

避免在一个短片里要求多个场景。
```

这张卡片可以作为速查图，也可以放进文章，或配一段社交媒体说明。

## 示例：上线检查清单卡片

```markdown
# 工具上线检查清单

发布前：

- 测试主要工具流程
- 检查移动端布局
- 审阅隐私说明
- 确认 sitemap 输出
- 跑一遍 smoke test

发布后：

- 提交 sitemap
- 检查 URL
- 观察用户反馈
```

这种卡片有用，是因为它把可重复流程变成了可以随时转发的提醒。

## 保持层级简单

只使用一个 H1 标题。多个 H1 会让卡片像几份文档拼在一起。

使用 H2 作为分节标题。它能提供结构，又不会让卡片显得过度设计。

使用列表帮助扫读。如果一个段落有太多信息，通常更适合改成列表。

加粗标签要克制。例如 `**目标：**` 能帮助识别字段，但整段加粗会显得嘈杂。

## 表格可以用，但要小

Markdown 表格适合紧凑对比：

```markdown
| 画幅 | 适合场景 |
| --- | --- |
| 横版 | 网站和 YouTube |
| 竖版 | Reels、Shorts、TikTok |
| 方形 | 信息流和缩略图 |
```

大表格在图片里很难读。如果表格列太多，建议改成分节说明，或直接写成文章。

## 代码块需要空间

短代码块可以很好看，但前提是不要太长。

好的例子：

````markdown
```js
const tool = "NavoKit";
console.log(tool);
```
````

如果代码很长，应该链接到完整源码。图片卡片更适合解释思路，而不是替代文档。

## 导出并复用

打开 [Markdown 转图片](/zh/tools/markdown-to-image)，粘贴 Markdown，并检查预览。如果卡片看起来太密，先改 Markdown，再导出。

导出后的图片可以用于：

- 社交媒体帖子
- Newsletter
- 内部文档
- 团队聊天
- 上线说明
- 教程线程
- 产品更新

如果需要配文，可以用 [AI 社交媒体文案生成器](/zh/tools/ai-social-booster) 起草第一版，然后改成你自己的表达。

## 隐私检查清单

分享导出图片前，确认它没有包含：

- API Key
- 密码
- 客户隐私
- 内部路线图
- 未公开客户材料
- 个人数据
- 未核对的 AI 输出

图片一旦发布，就可以像普通媒体文件一样被复制。发布前一定要检查。

## 最后检查

导出前问自己：

- 这张卡片是否只有一个清晰想法？
- 标题是否有用？
- 段落是否足够短？
- 离开上下文后是否仍然能看懂？
- 手机端是否可读？
- 是否包含隐私信息？
- 它是否更适合写成文章？

最好的 Markdown 图片卡片不是装饰品，而是让结构化文本更容易理解和传播。
