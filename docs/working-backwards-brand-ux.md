# NavoKit Working Backwards UX Brief

## Press release headline

NavoKit helps creators, developers, and marketers finish small digital tasks faster with focused AI tools.

## Customer promise

Open a tool. Finish a task. Leave with a usable output.

NavoKit should never feel like a directory, a hype page, or a gray-market service page. The product should feel like a calm workspace with a small number of reliable tools.

## Primary customers

1. Creators who need shareable assets and social drafts.
2. Developers who need clean exports for Markdown, code, and AI answers.
3. Marketers who need quick content variations and campaign assets.

## Jobs to be done

- Turn an idea into a usable content asset.
- Turn structured text into something shareable.
- Turn one prompt into a visual draft.
- Move from rough input to publishable output without installing heavy software.

## UX principles

1. Outcome before feature.
2. Tool above SEO copy.
3. One primary action per screen.
4. Calm monochrome brand; color only for function and state.
5. Clear limitations before trust is lost.
6. Expand by adding focused tools, not by becoming a cluttered directory.

## Homepage target

The homepage should answer four questions within the first scroll:

1. What is this? A lightweight AI toolbox.
2. Who is it for? Creators, developers, and marketers.
3. What can I finish here? Video, image export, social drafts.
4. Where do I click next? Featured tool or all tools.

## Future growth rule

New tools should enter the homepage through product content configuration:

- `homeFeatured`
- `homePriority`
- `homeAccent`
- `homePreviewTitle`
- `homePreviewText`
- `homePreviewMeta`

The homepage layout should not need code changes for every new tool.
